# stack-lifecycle-review

Use this skill to run quarterly GTM stack review and rationalization.

## Required Inputs

- current stack and usage data
- cost baseline
- performance outcomes
- incident logs and integration failures

## Review Policy

Every tool must be classified quarterly:

- retain
- optimize
- replace
- sunset

## Execution Sequence

1. Compare actual usage against intended scope.
2. Assess outcome contribution and reliability trend.
3. Re-score high-cost and low-use tools.
4. Identify consolidation opportunities.
5. Produce next-quarter action plan.

## Output Contract

Return:

- tool_status_matrix
- cost_vs_value summary
- replace_or_sunset candidates
- quarter_action_plan
- owner and due date for each action

## Failure Conditions

- no status classification for active tools
- no ownership for follow-up actions
- replacement proposed without migration step
